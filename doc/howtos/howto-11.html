boomerang howto #11:
Using the ads plugin

The ads plugin was devised as a simple way to check wether an ad blocking mechanism is in use and beacon the data back to the server

To achieve this you have to provide a url to an image that you konw is an ad and you know it will be there for a long time. It tries to create the image with the url you provide. If if succeeds the parameter ads is beaconed back with the value of 1, otherwise it is 0 

BOOMR.init({
  beacon_url: "http://localhost:8000",
  ads: {
    ads_src: "http://i.cdn.turner.com/cnn/.element/img/3.0/global/misc/advertisement.gif"
  }
});

There is only one parameter necessary for configuration

ads_src
[mandatory] The url to the source image to use for the test

One variable is added to the beacon

ads
It has a value of 1 is the test suceeded, meaning ads are working or a value of 0 is ads are disabled, by an adblocker for example.
